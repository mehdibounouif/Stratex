# Position Tracker Code Review

## Executive Summary
This position tracking system has a solid foundation but contains several **critical bugs** and areas needing improvement before production use. The most serious issues involve data integrity, error handling, and edge cases.

### 1. **Missing Persistence After add_position()**
**Location:** `add_position()`

**Problem:** After adding a position, the current positions are NOT saved to CSV. Only cash and history are saved.

**Impact:** If the system crashes after buying but before the next `update_prices()`, you'll lose track of your positions while cash is already deducted.

**Fix:**
```python
def add_position(self, ticker, quantity, entry_price, entry_date=None):
    # ... existing code ...
    
    self.cash -= position_total_cost
    if self.cash < 0:
        raise RuntimeError("Cash balance went negative — system invariant broken")
    
    self._save_cash()
    self._save_positions()  # ADD THIS LINE
    self._record_history()
    
    print(f"Bought {quantity} shares of {ticker} at ${entry_price} each. Cash remaining: ${self.cash:.2f}")
    return True
```

Add this method:
```python
def _save_positions(self):
    """Save current positions to CSV"""
    try:
        os.makedirs(os.path.dirname(self.positions_file), exist_ok=True)
        pd.DataFrame(self.positions).to_csv(self.positions_file, index=False)
    except Exception as e:
        print(f"Error saving positions: {e}")
```

### 3. **Floating Point Precision Issues**
**Location:** Multiple places (cash calculations, P&L)

**Problem:** Using floats for financial calculations can lead to rounding errors.

**Example:**
```python
>>> 0.1 + 0.2
0.30000000000000004
```

**Impact:** Small errors accumulate over many trades, leading to incorrect cash balances.

**Fix:** Use Python's `Decimal` for all financial calculations:

```python
from decimal import Decimal, ROUND_HALF_UP

class PositionTracker:
    def __init__(self, initial_capital=None):
        if initial_capital is None:
            initial_capital = TradingConfig.INITIAL_CAPITAL
        
        self.initial_capital = Decimal(str(initial_capital))
        self.cash = Decimal(str(initial_capital))
        # ...
    
    def add_position(self, ticker, quantity, entry_price, entry_date=None):
        entry_price = Decimal(str(entry_price))
        quantity = Decimal(str(quantity))
        # ...
```

### 5. **No Transaction Atomicity**
**Location:** `add_position()`, `remove_position()`

**Problem:** If an error occurs mid-transaction (e.g., after deducting cash but before saving), you're left in an inconsistent state.

**Fix:** Implement rollback logic:
```python
def add_position(self, ticker, quantity, entry_price, entry_date=None):
    # Save original state
    original_cash = self.cash
    original_positions = [p.copy() for p in self.positions]
    
    try:
        # ... perform transaction ...
        self.cash -= position_total_cost
        # ... update positions ...
        self._save_cash()
        self._save_positions()
        self._record_history()
        return True
    except Exception as e:
        # Rollback on error
        self.cash = original_cash
        self.positions = original_positions
        print(f"Transaction failed, rolled back: {e}")
        raise
```

### 14. **Add Position Validation on Load**

def _validate_positions(self):
    """Ensure loaded positions have required fields and valid values"""
    required_fields = ['ticker', 'quantity', 'entry_price', 'current_price']
    for pos in self.positions:
        for field in required_fields:
            if field not in pos:
                raise ValueError(f"Position missing required field: {field}")
        if pos['quantity'] <= 0:
            raise ValueError(f"Invalid quantity for {pos['ticker']}")
